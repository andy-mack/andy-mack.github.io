Git shows all text replaced in diff

When file was originally committed with line-ending different from operation system you work with, even few changes lead to all content replacement. Windows use both a carriage-return character and a linefeed character, whereas macOS and Linux systems use only the linefeed character.
Â 
One of the workaround is to temporary disable automatic line-ending replacement.
Check settings parameter (this command shows current value):
git config core.autocrlf

and set it temporary (before commit and before making changes) to false:
git config core.autocrlf true -> git config core.autocrlf false

Do not forget to change it back after pushing branch!
